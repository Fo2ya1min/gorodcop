# Журнал — сайт для постов через GitHub

Статичный сайт: читатели открывают страницу и видят записи без всякого входа.
Автор публикует новые записи прямо из браузера — они сохраняются как коммит
в `posts.json` этого репозитория. Никакого сервера и базы данных не нужно.

## Как это работает

- **Обычный читатель** — просто открывает сайт. Никакого логина.
- **Автор (админ)** — жмёт «Вход для автора» в углу сайта и вставляет свой
  личный токен GitHub (personal access token). После входа появляется форма
  для публикации и кнопки «Удалить» под каждой записью.
- Публикация = запись сохраняется через GitHub API прямо в файл `posts.json`
  этого репозитория. Токен хранится только в браузере автора (в
  `sessionStorage`, исчезает при закрытии вкладки) и никуда, кроме GitHub,
  не отправляется.

## Установка (10 минут)

### 1. Создайте репозиторий
Создайте **публичный** репозиторий на GitHub (например, `my-journal`) и
загрузите в него три файла из этой папки: `index.html`, `posts.json`,
`README.md`.

Репозиторий должен быть публичным — иначе читатели без входа не смогут
загрузить `posts.json` и увидеть записи.

### 2. Включите GitHub Pages
В репозитории: **Settings → Pages → Source** выберите ветку `main` и папку
`/ (root)`. Через минуту сайт появится по адресу вида:
`https://ваш-логин.github.io/my-journal/`

### 3. Настройте index.html
Откройте `index.html`, найдите в начале `<script>` блок `CONFIG` и впишите
свои данные:

```js
const CONFIG = {
  owner: "ваш-логин-на-github",
  repo: "my-journal",
  branch: "main",
  postsPath: "posts.json",
  siteTitle: "Название вашего журнала",
  siteTagline: "Короткое описание"
};
```

Сохраните, закоммитьте и запушьте изменение — сайт обновится сам.

### 4. Получите токен для входа как автор
1. Перейдите на GitHub: **Settings → Developer settings → Personal access
   tokens → Fine-grained tokens → Generate new token**.
2. **Repository access** — выберите «Only select repositories» и укажите
   именно этот репозиторий (не давайте доступ ко всем сразу).
3. **Permissions → Repository permissions → Contents** — выставьте
   **Read and write**.
4. Создайте токен и скопируйте его (он показывается один раз).
5. На сайте нажмите «Вход для автора» и вставьте токен.

Токен можно в любой момент отозвать в настройках GitHub — доступ автора
сразу пропадёт.

## Важно понимать

- Это полностью статичный сайт без сервера, поэтому «права доступа» держатся
  на владении токеном, а не на пароле или аккаунте — токен и есть ключ
  автора. Никому его не передавайте.
- После публикации новая запись обычно появляется у читателей в течение
  1–5 минут (это время кеширования GitHub для файла `posts.json`).
- Не публикуйте токен нигде, кроме поля входа на самом сайте — он не
  сохраняется в коде и не попадает в репозиторий.
- Если понадобится оформить сайт под себя — все стили в `index.html`,
  в блоке `<style>` вверху файла.
