# Журнал — сайт для постов через GitHub

Статичный сайт: читатели открывают страницу и видят записи без всякого входа.
Автор публикует новые записи прямо из браузера — они сохраняются как коммит
в `posts.json` этого репозитория.

## Как это работает

- **Читатель** — просто открывает сайт. Никакого логина.
- **Автор** — жмёт «Вход для автора» и вводит **пароль** (не токен).
- Сайт собирается и публикуется через **GitHub Actions**. При каждой сборке
  Action берёт из GitHub Secrets пароль и токен на запись, шифрует токен
  паролем (AES-256-GCM + PBKDF2, 250 000 итераций) и кладёт результат в
  файл `auth.json` в опубликованный сайт.
- Когда автор вводит пароль в браузере, JS на месте (Web Crypto API)
  пытается расшифровать `auth.json`. Если пароль верный — получается токен,
  которым сайт пишет в `posts.json` через GitHub API. Если пароль неверный —
  расшифровка технически проваливается (ошибка проверки подлинности AES-GCM).
- **Ни пароль, ни токен в открытом виде никогда не попадают в код,
  историю коммитов или логи Actions.** В файлах репозитория и в исходном
  `index.html` их просто нет.

## Важно понимать про безопасность

`auth.json` публично доступен на сайте (иначе браузер не смог бы его
прочитать), но токен внутри него зашифрован. Уровень защиты равен
стойкости вашего пароля к перебору "в лоб": используйте длинный
случайный пароль (например, `openssl rand -base64 24`), а не слово или
фразу. 250 000 итераций PBKDF2 делают перебор медленным, но не заменяют
хороший пароль.

Если пароль когда-либо скомпрометирован — сразу отзовите токен в GitHub
(Settings → Developer settings → Personal access tokens) и выпустите новый
с новым паролем (см. ниже).

## Установка

### 1. Создайте репозиторий и залейте файлы
Создайте **публичный** репозиторий на GitHub и загрузите в него всё
содержимое этой папки, включая скрытую папку `.github/`.

### 2. Настройте index.html
В блоке `CONFIG` внутри `index.html` укажите свой логин и имя репозитория:

```js
const CONFIG = {
  owner: "ваш-логин-на-github",
  repo: "имя-репозитория",
  branch: "main",
  postsPath: "posts.json",
  siteTitle: "Название вашего журнала",
  siteTagline: "Короткое описание"
};
```

### 3. Создайте токен на запись
**Settings → Developer settings → Personal access tokens → Fine-grained
tokens → Generate new token**:
- Repository access → Only select repositories → выберите именно этот репозиторий
- Permissions → Repository permissions → Contents → **Read and write**
- Создайте и скопируйте токен (показывается один раз)

### 4. Придумайте пароль
Например, сгенерируйте случайный: `openssl rand -base64 24`
(или любой другой генератор паролей). Сохраните его отдельно (менеджер
паролей) — восстановить забытый пароль нельзя, только сменить.

### 5. Добавьте секреты в репозиторий
**Settings → Secrets and variables → Actions → New repository secret**,
создайте два секрета:
- `WRITE_TOKEN` — токен из шага 3
- `SITE_PASSWORD` — пароль из шага 4

### 6. Включите GitHub Pages через Actions
**Settings → Pages → Source** — выберите **GitHub Actions** (не «Deploy
from a branch»).

### 7. Задеплойте
Закоммитьте и запушьте изменения в `main`. Workflow
`.github/workflows/deploy.yml` соберёт сайт и опубликует его на Pages —
адрес вида `https://ваш-логин.github.io/имя-репозитория/` появится во
вкладке Actions и в Settings → Pages. Готово: на сайте кнопка «Вход для
автора» теперь просит пароль.

## Как сменить пароль или токен

Просто обновите значения `SITE_PASSWORD` и/или `WRITE_TOKEN` в
Settings → Secrets, затем вручную запустите workflow (вкладка **Actions**
→ Deploy site → **Run workflow**) — новый `auth.json` перегенерируется
при следующей сборке.

## Прочее

- После публикации новая запись обычно появляется у читателей в течение
  1–5 минут (кеширование GitHub для `posts.json`).
- Стили — в `<style>` в начале `index.html`.
